CHANGES — cuda-kernel branch vs master
======================================
Branch  : cuda-kernel
Base    : master
Summary : GPU-accelerated triangle rendering via CUDA, plus logging,
          checkpointing, and improved stagnation control.


NEW FILES
---------

render.cu
    Replaces render.c with a CUDA-accelerated rasteriser. One GPU thread
    is launched per pixel; all threads in a block cooperatively load
    triangle data into shared memory before iterating over all N_TRIANGLES.
    Exposes the same render_triangles() signature as render.c so ga.c
    requires no changes, plus two lifecycle functions:
        cuda_renderer_init(w, h)  — allocates persistent GPU buffers once
        cuda_renderer_free()      — frees them at exit

CMakeLists.txt
    Added for CLion compatibility. Declares a mixed C + CUDA project
    targeting sm_89 (RTX 40xx).


MODIFIED FILES
--------------

triangle_ga.h
    - Added extern "C" guard so the header works in both gcc and nvcc
      translation units without linker errors.
    - Added cuda_renderer_init / cuda_renderer_free prototypes.
    - Added stagnation control defines:
        STAGNATION_RELATIVE  — toggle between relative (1) and absolute (0)
        STAGNATION_REL_TOL   — fractional improvement threshold (1e-4)
        STAGNATION_ABS_TOL   — raw improvement threshold (1e-7)

ga.c
    - Stagnation check now switches between relative and absolute mode
      based on STAGNATION_RELATIVE. (*****DEFAULT IS ABSOLUTE*****)
    - Added ga_save() / ga_load() for checkpoint support. ga_load()
      validates every fread return value and falls back to a fresh start
      if the file is corrupted.

main.c
    - Calls cuda_renderer_init / cuda_renderer_free around the GA run.
    - Opens run.log at startup; only generations where a genuine
      improvement occurred (stagnation_count == 0) are written to it.
    - Creates progress_ppm/ folder at startup; PPM snapshots are saved
      there every VISUALISE_EVERY (10000) generations.
    - Saves ga_checkpoint.bin at every snapshot and on clean exit.
      On startup, automatically resumes from the checkpoint if found.

Makefile
    - Switched to nvcc for compiling render.cu (explicit rule, no pattern
      ambiguity) and for linking.
    - Updated arch to sm_89, full nvcc path for WSL compatibility.
    - render.c removed from the build.
